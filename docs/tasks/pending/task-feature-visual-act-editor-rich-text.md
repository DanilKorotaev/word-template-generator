# Задача: Rich Text форматирование в визуальном редакторе актов

**Статус**: 📋 Запланировано  
**Приоритет**: 🟡 Средний  
**Категория**: Новые функции  
**Версия**: 2.1

## Контекст

Базовый визуальный редактор актов реализован в задаче  
`docs/tasks/done/task-feature-visual-act-editor.md`.

В текущей реализации текстовые поля передаются как plain text. Нужно добавить форматирование (жирный/курсив и т.д.) с корректной передачей в `docxtpl`.

## Цель

Сделать удобное редактирование rich text в Web UI и безопасную передачу форматирования в шаблоны Word через `docxtpl.RichText`.

## План работ

### Этап 1: UX редактора rich text
- [ ] Добавить панель форматирования для текстовых полей
- [ ] Поддержать как минимум: `bold`, `italic`
- [ ] Добавить режим переключения plain/rich для поля

### Этап 2: Формат данных
- [ ] Определить формат хранения разметки в `act.md`
- [ ] Не ломать обратную совместимость с существующими plain text полями
- [ ] Добавить миграционные/эвристические правила для старых актов

### Этап 3: Backend преобразование
- [ ] Реализовать преобразование разметки в `docxtpl.RichText`
- [ ] Добавить валидацию и безопасный fallback при некорректной разметке
- [ ] Покрыть edge-cases (переносы, смешанное форматирование, пустые блоки)

### Этап 4: Интеграция с шаблонами
- [ ] Документировать использование `{{r field_name}}` в шаблонах
- [ ] Поддержать комбинированные сценарии rich + token substitution
- [ ] Проверить совместимость с существующими шаблонами без rich text

### Этап 5: Тесты и документация
- [ ] Unit-тесты преобразования rich text
- [ ] Интеграционные тесты генерации DOCX
- [ ] Обновить `README.md` и `docs/USER_GUIDE_RU.md`

## Риски

- Усложнение шаблонов (`{{ ... }}` vs `{{r ... }}`)
- Потеря форматирования при round-trip сохранении
- Повышенная сложность сериализации и отладки

## Оценка

- **Сложность**: 🔴 Высокая
- **Время**: ~10-15 часов
