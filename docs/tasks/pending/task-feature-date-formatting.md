# Задача: Форматирование дат в полях актов

**Статус**: 📋 Запланировано  
**Приоритет**: 🟡 Средний  
**Категория**: Новые функции

## Описание

Нужна возможность указывать дату в полях акта числами (например, `04.03.2026`), а в итоговом Word-документе получать её в произвольном формате — например, «4 марта 2026 года» (месяц текстом на русском).

Дополнительно, нужна возможность использовать **текущую дату** (на момент генерации) без ручного ввода.

## Проблема

Сейчас все значения полей — простые строки. Если в поле написать `04.03.2026`, именно эта строка попадёт в шаблон. Чтобы получить «4 марта 2026 года», нужно вручную писать это целиком:

```yaml
поля:
  дата_начала: "4 марта 2026 года"
```

Это неудобно:
- При изменении даты нужно перепечатывать всю строку, а не поменять одну цифру
- Если одна и та же дата используется в разных форматах (`04.03.2026` в одном месте, `4 марта 2026 года` в другом) — приходится дублировать
- Нет способа автоматически подставить сегодняшнюю дату

## YAML и даты без кавычек

Важный нюанс: PyYAML `safe_load` по-разному обрабатывает форматы дат:

| Запись в YAML | Тип в Python | Пояснение |
|---|---|---|
| `дата: 2026-03-04` | `datetime.date(2026, 3, 4)` | YAML автоматически парсит ISO-формат |
| `дата: "2026-03-04"` | `str "2026-03-04"` | В кавычках — всегда строка |
| `дата: 04.03.2026` | `str "04.03.2026"` | Точки — не ISO, остаётся строкой |
| `дата: 04/03/2026` | `str "04/03/2026"` | Слэши — строка |

Вывод: даты можно писать без кавычек в обоих форматах. ISO-формат (`2026-03-04`) будет автоматически распознан YAML как объект даты — это нужно корректно обрабатывать в генераторе. Формат `04.03.2026` останется строкой — его нужно парсить самостоятельно.

## Решение

### 1. Расширение синтаксиса токенов: `[[поле|формат]]`

Расширить существующий синтаксис `[[token]]` для поддержки формата через разделитель `|`:

```yaml
поля:
  дата_начала: 04.03.2026
  дата_начала_текст: "[[дата_начала|d MMMM yyyy]]"
  дата_окончания: 2026-03-15
  дата_окончания_текст: "[[дата_окончания|dd.MM.yyyy]]"
```

Результат:
- `дата_начала_текст` → `"4 марта 2026"`
- `дата_окончания_текст` → `"15.03.2026"`

### 2. Форматы дат

Поддерживаемые спецификаторы (по аналогии со стандартными):

| Спецификатор | Описание | Пример |
|---|---|---|
| `d` | День без ведущего нуля | `4` |
| `dd` | День с ведущим нулём | `04` |
| `M` | Месяц число без нуля | `3` |
| `MM` | Месяц число с нулём | `03` |
| `MMM` | Месяц сокращённо | `мар` |
| `MMMM` | Месяц полностью (именительный) | `март` |
| `MMMMG` | Месяц полностью (родительный) | `марта` |
| `yy` | Год 2 цифры | `26` |
| `yyyy` | Год 4 цифры | `2026` |

Примеры форматирования:

| Формат | Результат |
|---|---|
| `d MMMMG yyyy` | `4 марта 2026` |
| `d MMMMG yyyy г.` | `4 марта 2026 г.` |
| `dd.MM.yyyy` | `04.03.2026` |
| `dd.MM.yy` | `04.03.26` |
| `d MMMM yyyy` | `4 март 2026` |
| `MMMM yyyy` | `март 2026` |

> **Почему `MMMMG`?** В русском языке даты используют родительный падеж месяца: «4 **марта**» (не «4 **март**»). Стандартный `MMMM` — именительный падеж, `MMMMG` (G = genitive) — родительный.

### 3. Текущая дата: `[[today]]` / `[[сегодня]]`

Встроенные токены, которые возвращают текущую дату на момент генерации:

```yaml
поля:
  дата_генерации: "[[сегодня|d MMMMG yyyy г.]]"
```

Результат (если генерация 4 марта 2026): `"4 марта 2026 г."`

Без формата `[[сегодня]]` вернёт дату в формате `dd.MM.yyyy` по умолчанию.

Можно также использовать напрямую как значение поля:

```yaml
поля:
  дата_генерации: сегодня
```

В этом случае `дата_генерации` станет текущей датой в формате `dd.MM.yyyy`.

### 4. Альтернатива: Jinja2-фильтр в шаблоне Word

Поскольку `docxtpl` использует Jinja2, можно зарегистрировать кастомный фильтр:

```
{{ дата_начала | date("d MMMMG yyyy") }}
```

Это позволяет форматировать дату прямо в шаблоне Word, а не в YAML. Оба подхода могут сосуществовать.

## Технические детали

### Парсинг дат

```python
import datetime
import re

MONTHS_NOM = [
    "январь", "февраль", "март", "апрель", "май", "июнь",
    "июль", "август", "сентябрь", "октябрь", "ноябрь", "декабрь",
]
MONTHS_GEN = [
    "января", "февраля", "марта", "апреля", "мая", "июня",
    "июля", "августа", "сентября", "октября", "ноября", "декабря",
]
MONTHS_SHORT = [
    "янв", "фев", "мар", "апр", "май", "июн",
    "июл", "авг", "сен", "окт", "ноя", "дек",
]

def parse_date(value) -> datetime.date | None:
    if isinstance(value, datetime.date):
        return value
    if not isinstance(value, str):
        return None
    for fmt in ("%d.%m.%Y", "%Y-%m-%d", "%d/%m/%Y"):
        try:
            return datetime.datetime.strptime(value, fmt).date()
        except ValueError:
            continue
    return None
```

### Формат-строка

```python
def format_date(dt: datetime.date, fmt: str) -> str:
    result = fmt
    result = result.replace("dd", f"{dt.day:02d}")
    result = result.replace("d", str(dt.day))
    # Порядок замен важен: MMMMG до MMMM, MMMM до MMM, MMM до MM, MM до M
    result = result.replace("MMMMG", MONTHS_GEN[dt.month - 1])
    result = result.replace("MMMM", MONTHS_NOM[dt.month - 1])
    result = result.replace("MMM", MONTHS_SHORT[dt.month - 1])
    result = result.replace("MM", f"{dt.month:02d}")
    result = result.replace("M", str(dt.month))
    result = result.replace("yyyy", str(dt.year))
    result = result.replace("yy", f"{dt.year % 100:02d}")
    return result
```

### Обновлённый regex для токенов

```python
TOKEN_RE = re.compile(r"\[\[([^\[\]\n|]+?)(?:\|([^\[\]\n]+?))?\]\]")
```

### Обратная совместимость

- Все существующие `[[токен]]` без `|` работают без изменений
- Поля без дат обрабатываются как раньше
- `datetime.date` из YAML конвертируется в `dd.MM.yyyy` строку до этапа токенов
- Формат `|` применяется только если значение распарсилось как дата; если нет — подставляется как строка

## Примеры использования

### Пример 1: Дата с текстовым месяцем

```yaml
---
поля:
  дата_начала: 04.03.2026
  дата_начала_текст: "[[дата_начала|d MMMMG yyyy г.]]"
---
```

Результат: `дата_начала_текст` = `"4 марта 2026 г."`

### Пример 2: Текущая дата

```yaml
---
поля:
  дата_генерации: "[[сегодня|d MMMMG yyyy г.]]"
---
```

Результат (4 марта 2026): `дата_генерации` = `"4 марта 2026 г."`

### Пример 3: Одна дата, несколько форматов

```yaml
---
поля:
  дата: 04.03.2026
  дата_цифрами: "[[дата|dd.MM.yyyy]]"
  дата_текстом: "[[дата|d MMMMG yyyy года]]"
  дата_короткая: "[[дата|dd.MM.yy]]"
---
```

Результат:
- `дата_цифрами` = `"04.03.2026"`
- `дата_текстом` = `"4 марта 2026 года"`
- `дата_короткая` = `"04.03.26"`

### Пример 4: ISO-дата из YAML (без кавычек)

```yaml
---
поля:
  дата: 2026-03-04
  дата_текстом: "[[дата|d MMMMG yyyy г.]]"
---
```

YAML автоматически распарсит `2026-03-04` в `datetime.date`. Генератор это обработает корректно.

### Пример 5: Текущая дата как значение поля

```yaml
---
поля:
  дата_сдачи: сегодня
---
```

Результат: `дата_сдачи` = `"04.03.2026"` (текущая дата в формате по умолчанию).

## Вопросы для проработки

- Нужен ли формат `d MMMMG yyyy` как формат по умолчанию для «текстовой» даты, или лучше оставить выбор пользователю?
- Нужно ли поддерживать вычисления с датами (например, `+7 дней`)? — скорее всего нет на первом этапе.
- Как обрабатывать невалидные даты (например, `32.13.2026`)? — выбрасывать `ValueError` с понятным сообщением.
