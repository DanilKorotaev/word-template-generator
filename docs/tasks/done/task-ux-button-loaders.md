# Задача: Лоадеры на кнопках при выполнении операций

**Статус**: ✅ Выполнено  
**Приоритет**: 🔴 Высокий  
**Категория**: Улучшения UX

## Описание

Добавлен визуальный фидбэк на async-кнопках: пользователь видит, что действие принято и выполняется.
Это убирает неопределенность при медленных операциях и снижает риск повторных нажатий.

## Что сделано

### 1. CSS для состояния loading

- [x] Добавлены стили `.loading` в `style.css`
- [x] Спиннер через `::after` псевдоэлемент
- [x] Анимация `spin` (`rotate(360deg)`)
- [x] Текст кнопки скрывается (`color: transparent`), размер кнопки сохраняется
- [x] Для `.secondary.loading` добавлен отдельный стиль спиннера

### 2. JS-обёртка `withLoading()`

- [x] Функция принимает кнопку (или Event) и async-функцию
- [x] Дизейблит кнопку, добавляет `.loading`
- [x] В `finally` восстанавливает состояние даже при ошибке
- [x] Добавлена защита от повторного вызова, если кнопка уже в loading

### 3. Обернуты async-кнопки

- [x] `pickFolder()` — «Выбрать папку...»
- [x] `loadActs()` — «Загрузить»
- [x] `validateWs()` — «Проверить»
- [x] `buildAll()` — «Сгенерировать все»
- [x] `buildOne()` — «Сгенерировать выбранный»

### 4. Синхронные кнопки

- [x] `useRecent()`, `removeRecent()`, `clearRecent()` оставлены без лоадера
- [x] `clearLog()` оставлена без лоадера

### 5. Обработка сетевых ошибок

- [x] В async-функции добавлена безопасная обработка сетевых ошибок
- [x] При сетевой ошибке показывается toast и пишется запись в лог
- [x] Убран риск `uncaught promise rejection` при недоступном сервере

## Технические детали

- Лоадер реализован на чистом CSS
- Добавлен универсальный helper `withLoading(...)`
- Добавлен helper `fetchJson(...)` для единообразной обработки сетевых сбоев
