# Задача: Визуальный редактор актов в Web UI (v2.0)

**Статус**: ✅ Выполнено  
**Приоритет**: 🟡 Средний  
**Категория**: Новые функции  
**Версия**: 2.0

## Что реализовано

### Этап 1: Инфраструктура
- [x] SPA-навигация (hash-based): `#generator` и `#editor`
- [x] Генератор вынесен в отдельный экран
- [x] Добавлена вкладка «Редактор актов»
- [x] Добавлен экран редактора

### Этап 2: Бэкенд API для редактора
- [x] `GET /api/act-data`
- [x] `POST /api/save-act`
- [x] `DELETE /api/delete-act`
- [x] `GET /api/template-variables`
- [x] `GET /api/project-fields`
- [x] YAML-сериализация + атомарная запись

### Этап 3: Список актов
- [x] UI-список с действиями «Редактировать», «Удалить», «Создать»
- [x] Подгрузка списка из `/api/acts`
- [x] Создание нового акта из UI
- [ ] Подтверждение удаления через отдельную модалку (пока `confirm`)

### Этап 4: Базовая форма редактора
- [x] Метаданные: шаблон, имя файла, номер (prefix + value)
- [x] Динамические поля с добавлением/удалением
- [x] Сохранение через `/api/save-act`
- [ ] Drag & drop сортировка полей (опционально, не реализовано)

### Этап 5: Типы полей
- [x] Типы: `text`, `multiline`, `date`, `number`
- [x] Поддержка многострочного ввода
- [x] Ввод даты с календарем, форматом и превью
- [x] Временная стабилизация UX: `reference` отключен в выборе типа до доработки

### Этап 6: Подсказки из шаблона
- [x] Загрузка переменных шаблона через `/api/template-variables`
- [x] Индикаторы заполнено/пусто
- [x] Кнопка «Добавить недостающие поля»

### Этап 7: Наследование `project.md`
- [x] Загрузка полей проекта через `/api/project-fields`
- [x] Отображение наследуемых полей
- [x] Переопределение значений на уровне акта

### Этап 8: Генерация из редактора
- [x] Кнопка «Сгенерировать»: сначала save, затем `build-one`
- [x] Отображение результата в редакторе
- [x] Открытие результата через `/api/open-file`

## Дополнительно реализовано

- [x] Улучшена верстка полей редактора
