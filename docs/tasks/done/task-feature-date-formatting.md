# Задача: Форматирование дат в полях актов

**Статус**: ✅ Выполнено  
**Приоритет**: 🟡 Средний  
**Категория**: Новые функции

## Что реализовано

### 1. Локальный модуль форматирования дат

- [x] Добавлен `src/word_template_generator/utils/date_format.py`
- [x] `parse_date(value)` поддерживает:
  - `datetime.date` / `datetime.datetime`
  - `dd.MM.yyyy`
  - `yyyy-MM-dd`
  - `dd/MM/yyyy`
  - `today` / `сегодня`
- [x] `format_date(dt, fmt)` поддерживает:
  - `d`, `dd`
  - `M`, `MM`, `MMM`, `MMMM`, `MMMMG`
  - `yy`, `yyyy`
- [x] Значение по умолчанию для даты без формата: `dd.MM.yyyy`
- [x] Невалидные даты вызывают `ValueError` с понятным сообщением

### 2. Токены `[[поле|формат]]` в генераторе

- [x] Обновлен regex токенов: `[[token]]` и `[[token|format]]`
- [x] При `|format` значение токена форматируется как дата, если распознается
- [x] Для недатных значений с форматом сохраняется исходная строка
- [x] Обратная совместимость сохранена

### 3. Встроенные токены `[[today]]` / `[[сегодня]]`

- [x] Добавлены встроенные токены текущей даты
- [x] Работают и с форматом (`[[today|d MMMMG yyyy]]`), и без (`[[today]]`)

### 4. Поля-значения `today` / `сегодня`

- [x] Если значение поля равно `today` или `сегодня`, оно преобразуется в текущую дату
- [x] Преобразование выполняется до этапа токенов

### 5. Обработка дат из YAML

- [x] `datetime.date` из YAML (`2026-03-04` без кавычек) корректно обрабатывается
- [x] Без формата выводится в `dd.MM.yyyy` вместо `YYYY-MM-DD`

### 6. Тесты

  - `tests/test_date_format.py`

- [x] Покрыты сценарии: парсинг, форматирование, today-токены, ошибки невалидных дат

## Принятые решения по вопросам

- [x] Формат `d MMMMG yyyy` **не** используется как формат по умолчанию.  
      По умолчанию остается `dd.MM.yyyy`; текстовый формат задается явно через `|...`.
- [x] Невалидные даты (например, `32.13.2026`) приводят к `ValueError` с понятным сообщением.
