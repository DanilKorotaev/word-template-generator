# Задача: Интерактивный список сгенерированных файлов

**Статус**: ✅ Выполнено  
**Приоритет**: 🔴 Высокий  
**Категория**: Улучшения UX

## Описание

После генерации добавлена секция «Результаты» со списком созданных файлов.
Теперь файл можно открыть прямо из UI: по кнопке «Открыть» или двойным кликом по строке.

## Что сделано

### 1. Расширен ответ API генерации

- [x] Обновлены `build-all` и `build-one` в `routes.py`: возвращают список `files`
- [x] Формат элемента файла: `{"act": str, "path": str, "name": str}`
- [x] Путь возвращается как абсолютный

### 2. Добавлен эндпоинт `/api/open-file`

- [x] Добавлен `OpenFilePayload` в `schemas.py` с полем `path: str`
- [x] Реализовано кроссплатформенное открытие: `os.startfile` (Windows), `open` (macOS), `xdg-open` (Linux)
- [x] Добавлена валидация: файл должен существовать и иметь расширение `.docx` или `.docm`
- [x] Возвращается понятная ошибка при невозможности открытия

### 3. Добавлена UI секция «Результаты»

- [x] Добавлен блок `resultsCard` в `index.html` (по умолчанию скрыт)
- [x] Добавлены стили `.results-list`, `.result-item` и связанных элементов
- [x] Элемент содержит иконку, имя файла, полный путь и кнопку «Открыть»
- [x] Поддержан двойной клик по строке для открытия файла

### 4. Обновлена JS-логика

- [x] После успешной генерации используется `data.files` для рендера
- [x] Добавлена `renderResults(files)` для заполнения `#resultsList`
- [x] Добавлена `openFile(path)` с POST-запросом в `/api/open-file`
- [x] При новой генерации список результатов очищается и заполняется заново
- [x] Если `files` пуст, секция автоматически скрывается

### 5. Очистка и состояние

- [x] Добавлена кнопка «Очистить» в секции результатов (`clearResults()`)
- [x] Видимость секции не сбрасывается при смене workspace через загрузку актов
