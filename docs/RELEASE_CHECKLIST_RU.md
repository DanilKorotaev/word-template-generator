# Release Checklist (RU)

## 1) Подготовка перед релизом

- [ ] Обновлены `README.md` и docs на русском.
- [ ] Проверены сценарии из `docs/USER_GUIDE_RU.md`.
- [ ] Проверены one-click скрипты `start.command` и `start.ps1`.
- [ ] Проверено хранение недавних проектов в Web UI.

## 2) Функциональный smoke-тест

- [ ] `aosr-gen web-ui` запускается и открывает браузер.
- [ ] `Загрузить акты` показывает список актов.
- [ ] `Проверить` проходит на валидном workspace.
- [ ] `Сгенерировать все` создаёт DOCX в `generated/`.
- [ ] `Сгенерировать выбранный` корректно пересобирает один акт.
- [ ] После перезапуска браузера восстанавливается последний workspace.

## 3) Негативные кейсы

- [ ] Неверный путь `workspace` показывает понятную ошибку.
- [ ] Отсутствующий `template.docx` показывает понятную ошибку.
- [ ] Битый front matter даёт явное сообщение в логе.

## 4) Техпроверка

- [ ] `python -m compileall src` проходит без ошибок.
- [ ] Нет случайных секретов и лишних файлов в изменениях.
- [ ] Проверены только целевые файлы проекта (без `.venv`).

## 5) Стратегия коммитов

Рекомендуемая последовательность:

1. `docs`: русская документация и инструкции эксплуатации.
2. `feat(web-ui)`: recent projects + UX-полировка local-first.
3. `chore`: one-click скрипты и инфраструктурные шаблоны (Docker/Nginx, если нужны в релизе).

## 6) Критерий готовности

Пользователь без IT-навыков проходит сценарий по `docs/USER_GUIDE_RU.md` самостоятельно, без ручных команд в терминале.
